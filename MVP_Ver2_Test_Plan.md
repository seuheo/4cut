# MVP Ver2 테스트 계획

## 📋 테스트 목표

MVP Ver2에서 추가된 Phase 1 (JSON 프레임 시스템)과 Phase 2 (동영상 생성 및 공유) 기능이 정상적으로 작동하는지 확인합니다.

---

## 🧪 Phase 1: JSON 프레임 시스템 테스트

### 테스트 1.1: JSON 파일 로드 확인
**목표**: `frames.json` 파일이 정상적으로 로드되고 Frame 객체에 슬롯 정보가 병합되는지 확인

**절차**:
1. 앱 실행
2. FrameRepository 초기화 확인
3. AppNavigation에서 `loadSlotsFromJson(context)` 호출 확인
4. 로그에서 "JSON에서 X개의 프레임 슬롯 정보 로드 완료" 메시지 확인

**예상 결과**:
- `frames.json` 파일이 정상적으로 읽혀짐
- 3개 프레임 (`image_e15024`, `long_form_white`, `long_form_black`)의 슬롯 정보가 로드됨
- Frame 객체에 `slots` 필드가 정상적으로 병합됨

**검증 방법**:
- Logcat에서 "FrameRepository: JSON에서 3개의 프레임 슬롯 정보 로드 완료" 확인
- `FrameRepository.getFrameById("image_e15024")?.slots`가 null이 아님 확인

---

### 테스트 1.2: JSON 슬롯 정보를 사용한 이미지 합성
**목표**: JSON 슬롯 정보를 사용하여 이미지가 정상적으로 합성되는지 확인

**절차**:
1. 갤러리에서 사진 4장 선택
2. `image_e15024` 프레임 선택
3. "인생네컷 만들기" 버튼 클릭
4. 이미지 합성 실행

**예상 결과**:
- ImageComposer가 JSON 슬롯 정보를 사용하여 합성 수행
- 로그에서 "JSON 슬롯 정보 사용: 4개" 메시지 확인
- 합성된 이미지가 정상적으로 생성됨
- 사진이 프레임 슬롯에 올바르게 배치됨

**검증 방법**:
- Logcat에서 "JSON 슬롯 정보 사용: 4개" 확인
- 미리보기 다이얼로그에서 사진이 올바른 위치에 배치되었는지 시각적 확인
- 합성된 이미지 저장 후 갤러리에서 확인

---

### 테스트 1.3: 기존 프레임 하위 호환성 확인
**목표**: 슬롯 정보가 없는 기존 프레임이 정상적으로 작동하는지 확인

**절차**:
1. 갤러리에서 사진 4장 선택
2. `single_frame` 프레임 선택 (슬롯 정보 없음)
3. "인생네컷 만들기" 버튼 클릭
4. 이미지 합성 실행

**예상 결과**:
- ImageComposer가 기존 하드코딩 로직을 사용하여 합성 수행
- 로그에서 "기존 하드코딩 로직 사용" 메시지 확인 (또는 JSON 슬롯 정보 사용 메시지 없음)
- 합성된 이미지가 정상적으로 생성됨

**검증 방법**:
- Logcat에서 JSON 슬롯 정보 사용 메시지 없음 확인
- 합성된 이미지 저장 후 갤러리에서 확인

## 🎬 Phase 2: 동영상 생성 및 공유 테스트

### 테스트 2.1: 슬라이드쇼 동영상 생성
**목표**: 원본 사진 4장으로 슬라이드쇼 MP4 동영상이 정상적으로 생성되는지 확인

**절차**:
1. 갤러리에서 사진 4장 선택
2. 프레임 선택 및 이미지 합성
3. "저장" 버튼 클릭
4. 갤러리 저장 완료 대기
5. 데이터베이스 저장 및 동영상 생성 확인

**예상 결과**:
- VideoSlideShowCreator.createSlideShowVideo()가 호출됨
- 로그에서 "JCodec 동영상 생성 시작" 메시지 확인
- 동영상 파일이 `externalFilesDir/videos/` 경로에 생성됨
- PhotoEntity에 `videoPath` 필드가 정상적으로 저장됨

**검증 방법**:
- Logcat에서 "JCodec 동영상 생성 완료: [경로]" 확인
- 파일 시스템에서 동영상 파일 존재 확인
- 데이터베이스에서 PhotoEntity.videoPath 필드 확인

---

### 테스트 2.2: 동영상 파일 형식 및 품질 확인
**목표**: 생성된 동영상이 올바른 형식과 품질을 가지는지 확인

**절차**:
1. 테스트 2.1 완료 후 생성된 동영상 파일 확인
2. 동영상 재생 확인

**예상 결과**:
- MP4 형식 파일
- 해상도: 1080x1920 (인스타그램 스토리 최적화)
- 각 사진당 2초 표시 (총 8초)
- 프레임 레이트: 30fps
- 재생 가능한 상태

**검증 방법**:
- 파일 확장자 `.mp4` 확인
- 동영상 플레이어로 재생하여 해상도 및 길이 확인
- 각 사진이 2초씩 표시되는지 확인

---

### 테스트 2.3: PhotoDetailScreen에서 동영상 공유
**목표**: PhotoDetailScreen에서 이미지와 동영상을 함께 공유할 수 있는지 확인

**절차**:
1. 홈 화면에서 저장된 사진 선택
2. PhotoDetailScreen으로 이동
3. Share 아이콘 클릭
4. Android 네이티브 공유 시트 확인

**예상 결과**:
- Share 아이콘이 표시됨
- Share 아이콘 클릭 시 Android 네이티브 공유 시트가 표시됨
- 이미지와 동영상이 함께 공유 가능한 상태

**검증 방법**:
- 공유 시트에서 이미지와 동영상 파일이 포함되어 있는지 확인
- 공유 대상 앱 선택 시 정상적으로 공유되는지 확인

---

### 테스트 2.4: 이미지만 있는 사진 공유 (하위 호환성)
**목표**: 동영상이 없는 기존 사진도 정상적으로 공유되는지 확인

**절차**:
1. 동영상 생성 이전에 저장된 사진 선택 (또는 동영상 생성 실패한 사진)
2. PhotoDetailScreen으로 이동
3. Share 아이콘 클릭

**예상 결과**:
- Share 아이콘이 정상적으로 작동
- 이미지만 공유됨 (동영상 없음)

**검증 방법**:
- 공유 시트에서 이미지만 포함되어 있는지 확인
- 공유 대상 앱 선택 시 이미지가 정상적으로 공유되는지 확인

## 🚨 알려진 이슈 및 해결 방안

### 이슈 1: JSON 파일 로드 실패
**증상**: 로그에 "JSON 로드 실패" 메시지 표시
**해결 방안**:
- `frames.json` 파일이 `app/src/main/assets/` 경로에 존재하는지 확인
- JSON 형식이 올바른지 확인
- 앱 재설치

### 이슈 2: 동영상 생성 실패
**증상**: 동영상 파일이 생성되지 않음
**해결 방안**:
- JCodec 라이브러리 의존성 확인
- 저장소 권한 확인
- 로그에서 오류 메시지 확인

### 이슈 3: 공유 기능 실패
**증상**: Share 아이콘 클릭 시 공유 시트가 표시되지 않음
**해결 방안**:
- FileProvider 설정 확인 (`AndroidManifest.xml`, `provider_paths.xml`)
- 파일 경로가 올바른지 확인
- 로그에서 오류 메시지 확인
