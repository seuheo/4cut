## 🧪 **Phase 1: 핵심 기능 및 예외 처리 테스트**

### **목표**: 앱의 핵심 기능이 정상 작동하는지 확인하고, 비정상적인 상황에서 앱이 강제 종료되지 않고 사용자에게 명확한 피드백을 주는지 검증합니다.

### **Test Case 1.1: 주요 기능 검증 (Happy Path) ✅**

#### **테스트 시나리오**:
1. 앱 실행 → 홈 화면 진입
2. 하단 네비게이션 바에서 "Frame" 탭 클릭
3. 사진 4장 선택 (갤러리에서 또는 랜덤 사진 추가)
4. "다음 단계" 버튼 클릭
5. 프레임 선택 화면에서 원하는 프레임 선택
6. 프레임 선택 후 결과 화면으로 이동
7. "저장하기" 버튼 클릭
8. 저장 완료 메시지 확인

#### **예상 결과**:
- ✅ 모든 기능이 의도한 대로 작동
- ✅ 최종 결과물(적용된 프레임 이미지)이 정상적으로 생성
- ✅ 이미지가 갤러리에 정상적으로 저장됨
- ✅ 성공 메시지 표시

#### **현재 상태**: ✅ 테스트 진행 가능

---

### **Test Case 1.2: 예외 처리 검증 (에러 메시지 및 화면 복귀)**

#### **테스트 시나리오**:
1. **저장 공간 부족 시뮬레이션**: 
   - 미리 가득 찬 스토리지로 에뮬레이터 설정 후 저장 시도
   - 예상 결과: 에러 메시지 표시 + 홈 화면 복귀

2. **권한 미허용 테스트**:
   - 갤러리 접근 권한 거부
   - 갤러리에서 사진 선택 시도
   - 예상 결과: 권한 요청 안내 메시지 + 권한 설정 유도

3. **네트워크 연결 끊김** (서비스 기능 적용 시):
   - 항공 모드 활성화 후 네트워크 기능 사용
   - 예상 결과: 네트워크 오류 메시지 + 이전 화면 복귀

4. **이미지 합성 중 뒤로 가기**:
   - 이미지 합성 프로세스 중 뒤로 가기 버튼 클릭
   - 예상 결과: 프로세스 중단 메시지 + 이전 화면 복귀

5. **프레임을 선택하지 않고 다음 단계 진행 시도**:
   - 사진만 선택하고 프레임 선택 없이 "다음 단계" 클릭
   - 예상 결과: 버튼 비활성화 상태 유지

#### **예상 결과**:
- ✅ 앱이 강제 종료되지 않음
- ✅ 사용자 친화적인 에러 메시지 표시 (Snackbar, Toast 등)
- ✅ 이전 화면이나 안전한 홈 화면으로 복귀
- ✅ 앱을 계속 사용 가능한 상태 유지

#### **현재 상태**: 🟡 테스트 진행 중 (저장 기능 등 일부 시나리오 추가 구현 필요)

## 🚂 **Phase 2: KTX 역 목록 데이터 무결성 테스트**

### **목표**: 'KTX 역 선택' 기능과 '캘린더 탭' 간의 데이터가 일관성 있게 연동되는지 검증합니다.

### **Test Case 2.1: 캘린더 탭 노선 필터 검증 ✅**

#### **테스트 시나리오**:
1. CalendarScreen에 진입
2. 노선 필터 버튼 확인
3. '경부선' 선택
4. '호남선' 선택

#### **예상 결과**:
- ✅ 필터 옵션으로 '경부선', '호남선'만 표시됨
- ✅ '경부선' 선택 시 경부선 관련 KTX 역만 표시됨
- ✅ '호남선' 선택 시 호남선 관련 KTX 역만 표시됨

#### **현재 상태**: ✅ 구현 완료, 테스트 진행 가능

---

### **Test Case 2.2: 'KTX 역 선택' 목록 검증 ✅**

#### **테스트 시나리오**:
1. FrameScreen 또는 PhotoDetailScreen 진입
2. 'KTX 역 선택 (선택사항)' 드롭다운 또는 리스트 확인
3. 표시되는 모든 역 목록 수집

#### **예상 결과**:
- ✅ FrameApplyScreen의 역 목록이 경부선 + 호남선만 포함
- ✅ CalendarScreen의 필터 목록과 완전히 일치

#### **현재 상태**: ✅ 구현 완료, 테스트 진행 가능

---

### **Test Case 2.3: 노선-역 연동 검증 (Cross-Check) ✅**

#### **테스트 시나리오**:
1. CalendarScreen에서 '경부선' 선택 시 나타나는 역 목록 확인
2. 'KTX 역 선택' 화면의 경부선 역 목록과 대조
3. 동일하게 호남선도 대조

#### **예상 결과**:
- ✅ 두 기능 간의 데이터가 100% 일치
- ✅ 누락되거나 추가된 역이 없음

#### **현재 상태**: ✅ 구현 완료, 테스트 진행 가능

## 📱 **Phase 3: 공유하기 (SNS 연동) 기능 테스트**

### **목표**: 앱 내에서 생성된 결과물(사진)이 외부 SNS 앱과 정상적으로 연동되는지 검증합니다.

### **Test Case 3.1: 공유 기능 활성화 검증 🟡**

#### **테스트 시나리오**:
1. 결과물 확인 화면에서 '공유하기' 아이콘/버튼 클릭

#### **예상 결과**:
- ✅ Android OS의 기본 공유 시트(Share Sheet)가 정상적으로 실행됨

#### **현재 상태**: 🟡 기본 공유 기능은 구현되어 있음 (Instagram Story 직접 공유 기능 추가 필요)

---

### **Test Case 3.2: 인스타그램 연동 테스트**

#### **테스트 시나리오**:
1. 공유 시트에서 '인스타그램' (스토리) 선택
2. 공유 대상(채팅방)을 선택
3. 이미지 전송 확인

#### **예상 결과**:
- ✅ 인스타그램 앱이 실행됨
- ✅ 이미지가 인스타그램 스토리 편집 화면에 정상적으로 로드됨
- ✅ 업로드가 가능함

#### **현재 상태**: ✅ 구현 완료 (Instagram Story Intent)

---

### **Test Case 3.3: 기타 앱 연동 테스트**

#### **테스트 시나리오**:
1. 공유 시트에서 '카카오톡', '메시지', '이메일' 등 다른 공유 옵션 선택
2. 이미지 전송 확인

#### **예상 결과**:
- ✅ 이미지가 해당 앱으로 정상적으로 전달(첨부)됨

#### **현재 상태**: ✅ 기본 공유 기능으로 모든 앱 지원

## 📱 **Phase 4: 실제 디바이스 사용성 (QA) 테스트**

### **목표**: Android Studio의 에뮬레이터가 아닌, 실제 스마트폰에서 터치 인터랙션과 UI가 정상적으로 작동하는지 최종 검증합니다.

### **Test Case 4.1: 터치 및 제스처 반응 검증 🟡**

#### **테스트 시나리오**:
- **버튼 클릭**: 모든 버튼을 손가락으로 클릭하여 반응 확인
- **스크롤**: PhotoGrid, CalendarScreen 등 목록 화면 스크롤 확인
- **스와이프**: FrameCarousel 같은 캐러셀 UI 좌우 스와이프 제스처 확인

#### **예상 결과**:
- ✅ 모든 버튼이 정확한 터치 영역을 가지며 즉각적으로 반응
- ✅ 스크롤이 부드럽게 작동
- ✅ 캐러셀 UI가 좌우 스와이프에 정확히 반응

#### **현재 상태**: 🟡 실제 기기 테스트 필요

---

### **Test Case 4.2: 기기별 UI 레이아웃 검증 🟡**

#### **테스트 시나리오**:
1. 해상도가 다른 1~2개 이상의 실제 기기에서 앱 실행
2. UI 레이아웃 점검

#### **예상 결과**:
- ✅ 텍스트가 잘리지 않음
- ✅ 버튼이 화면 밖을 벗어나지 않음
- ✅ 이미지가 겹치지 않음

#### **현재 상태**: 🟡 실제 기기 테스트 필요

---

### **Test Case 4.3: 앱 성능 체감 테스트 🟡**

#### **테스트 시나리오**:
1. 앱 실행 속도 측정
2. 화면 전환 속도 체감
3. 이미지 로딩 속도 체감

#### **예상 결과**:

- ✅ 이미지 처리 시 과도한 지연 없음

#### **현재 상태**: 🟡 실제 기기 테스트 필요
